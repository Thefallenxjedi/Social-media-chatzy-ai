# Chatzy AI Image & Visual Style Guide

Based on the provided graphic assets, here is the breakdown of Chatzy's visual identity and how images should be constructed. Use this guide to maintain a consistent, high-quality, modern SaaS aesthetic across all LinkedIn graphics and marketing materials.

## 1. Core Brand Elements

* **Logo:** The Chatzy logo consists of a friendly, pill-shaped robot face outline (with closed, smiling eyes) next to the word "Chatzy" in a clean sans-serif font.
* **Primary Color:** Vibrant Indigo / Purple-Blue (approx. `#4F46E5`). This is used for the logo, primary buttons, borders, and hand-drawn accents.
* **Background Colors:** predominantly clean White (`#FFFFFF`) or very soft tinted off-whites/light purples for card backgrounds to create depth without clutter.
* **Gradients:** Smooth, vibrant gradients are used for high-impact announcement cards (e.g., Purple-to-Blue for Chatzy, Teal-to-Green for partners like Cal.com).

## 2. Typography & Copy

* **Font:** Clean, modern, geometric sans-serif (e.g., Inter, Roboto, or Plus Jakarta Sans).
* **Headlines:** Bold and highly legible. Often using color to emphasize key words (e.g., "Your **WhatsApp**. Your **AI Agent**.").
* **Text Density:** Extremely low. Images use brief phrases, bullet points, or short descriptive titles ("How it works", "Live Conversations"). 
* **Bullet Points:** Represented by clean, circular icons (often purple outlines or green filled circles with white checkmarks).

## 3. Design Aesthetic & Layout (The "Bento Box" UI Style)

Chatzy graphics heavily utilize a modular, "Bento Box" layout style, which makes complex software look simple and approachable.

* **Cards and Modules:** Content is separated into distinct sections using white or light-tinted cards with soft, large rounded corners and very subtle drop shadows.
* **UI Mockups:** Instead of messy screenshots, the graphics use abstracted UI components. You will see simplified chat interfaces, toggles (green for active), dropdowns, and settings panels.
* **Flowcharts & Processes:** "How it works" sections are visualized vertically or horizontally using icons inside rounded squares, connected by delicate, dotted purple arrows pointing to the next step.
* **Hand-drawn Accents:** Occasionally, a hand-drawn purple arrow is used to draw attention from a value proposition text box to the UI element that solves it (e.g., pointing from a prompt to the agent generation flow).

## 4. Common Image Formats & Templates

### Format A: The Product Breakdown (UI + Flow)
**Use for:** Explaining a complex feature (Multi-Agent Routing, Agent Creation).
* **Left side:** Main headline, a highlighted value prop box, and 3 checkmarked benefits.
* **Center:** A vertical 3- or 4-step flowchart with icons and dotted arrows ("How it works").
* **Right side:** An abstracted UI mockup showing the end result (e.g., a list of active agents, or a settings panel with integrations).

### Format B: The Integration / Announcement Card
**Use for:** Announcing new integrations (e.g., Cal.com) or major updates.
* **Top Half:** Split into two vibrant, gradient-filled squares with rounded corners. Left square contains the Chatzy logo; right square contains the partner logo.
* **Bottom Half:** A full-width, dark contrasting bar (navy or black) with a single line of bold white text (e.g., "Integration with Cal.com is live").

### Format C: The Platform Feature (Visual + Benefits)
**Use for:** Highlighting a specific channel or capability (e.g., WhatsApp integration).
* **Left side:** Bold, multi-colored headline + 3 to 4 checkmark bullet points.
* **Right side:** A large, recognizable icon of the platform (e.g., WhatsApp) connected via a dotted line to a friendly Chatzy robot avatar icon.

## 5. Key Rules for Creation

1. **Whitespace is crucial:** Do not clutter the image. Give every card and text block plenty of room to breathe.
2. **Keep UI elements abstract:** Don't use actual messy browser screenshots. Recreate the UI elements cleanly in Figma/Illustrator.
3. **Consistent Corner Radii:** Use the same rounded corners on all boxes, buttons, and backgrounds to maintain the modern SaaS look.
4. **Friendly, not robotic:** Always use the smiling robot icon and soft color palettes to make the AI feel accessible and helpful, rather than intimidating.
